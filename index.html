  1. Block, Inline, Inline-block elements
    • Block Elements => the elements who cover full width available and no elements can sit beside
      <address>  <article>  <aside>  <blockquote>  <canvas>  <dd>  <div>  <dl>  <dt>  <fieldset>  <figcaption>  <figure>  <footer>  <form>  <h1>....<h6>  <header>  <hr>  <li>  <main>
<nav>  <noscript>  <ol>  <p>  <pre>  <section>  <table>  <tfoot>

      CSS Style:

      display: block;

    • Inline Elements => the elements cover only necessary width and other inline elements can sit beside
      <a>  <abbr>  <acronym>  <b>  <bdo>  <big>  <br>  <button>  <cite>  <code>  <dfn>  <em>  <i>  <img>  <input>  <kbd>  <label>  <map>  <object>  <output>  <q>  <samp>  <script>
<select>  <small>  <span>  <strong>  <sub>  <sup>  <textarea>  <time>  <tt>  <var>

      CSS Style:

      display: inline;
    • Inline-block i.e. display: inline-block; is not assigned to any elements. We can use this on our needs. So, that element will treat like both inline and block style i.e. takes necessary width but height will be block.
